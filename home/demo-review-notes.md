# Demo review notes

## Assumptions

- This is a first-draft demo, not a full migration.
- Source docs were partially inaccessible to direct scraping, so the demo mirrors the indexed IA and representative page topics.
- API reference pages are intentionally summarized. If Eseye can provide OpenAPI specs, the Developer APIs space should switch to GitBook's OpenAPI-generated reference model.
- The demo is staged in the requested GitBook org and uses a URL-driven adaptive view for AT&T.

## Feedback that would help most

1. Which section should be the hero of the sales conversation: Infinity, AnyNet+, SMARTconnect, Hera routers, or APIs?
2. Should the landing page lean more technical, more commercial, or more support-oriented?
3. Is there an OpenAPI spec or source repo available for the API docs?
4. Which pages should become authenticated customer-only material in a second pass?

## Adaptive demo view

Use the AT&T demo URL to hide the Developer APIs section from the top navigation. The default view still shows the full site.
